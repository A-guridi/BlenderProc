class src.main.Module.Module(config)[source]¶

Configuration:

All of these values can be set per Module or of the global config defined in the main.Initializer:

“module”: “main.Initializer”, “config”:{

“global”: {
“output_dir”: “<args:X>”

}

}

If they are set globally all modules will inherit them, if there is no module defined key available.

Parameter Description
output_is_temp If True, all files created in this module will be written into the temp_dir. If False,
the output_dir is used. Type: bool.  
output_dir The path to a directory where all persistent output files should be stored. If it doesn’t exist,
it is created automatically. Type: string. Default: “.

Checks if the given output entry already exists in the list of outputs, by checking on the key and path. Also throws an error if it detects an entry having the same key but not the same path and vice versa since this is ambiguous.

Parameters:

_register_output(default_prefix, default_key, suffix, version, stereo=False, unique_for_camposes=True, output_key_parameter_name='output_key', output_file_prefix_parameter_name='output_file_prefix')[source]¶

Registers new output type using configured key and file prefix.

Parameters:
  • default_prefix – The default prefix of the generated files.
  • default_key – The default key which should be used for storing the output in merged file.
  • suffix – The suffix of the generated files.
  • version – The version number which will be stored at key_version in the final merged file.
  • stereo – Boolean indicating whether the output of this rendering result will be stereo.
  • unique_for_camposes – True if the output to be registered is unique for all the camera poses
  • output_key_parameter_name – The parameter name to use for retrieving the output key from the config.
  • output_file_prefix_parameter_name – The parameter name to use for retrieving the output file prefix from the config.
